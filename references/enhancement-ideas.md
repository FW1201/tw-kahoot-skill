# 格式完整度 backlog（v0.1.0，全新 repo）

本 skill 從第一版就採用純格式產生器策略（比照其他四個 tw-*-skill 已完成的縮減方向），沒有經歷過瀏覽器自動化再拿掉的過程，因此沒有「已隨自動化層移除」這一節。

## 下一輪參考

1. **取得更新版官方範本確認**：目前欄位配置依據 2019 年的官方範本檔案（見 `references/platform-features.md`），是找得到的最新第一方託管檔案，但無法排除目前 app 內即時下載的範本已再更新。下一輪若有 Kahoot 帳號/瀏覽器工具，應實際走一次 Import spreadsheet UI 流程，下載當下的官方範本重新比對欄位/字元上限/時間檔位。
2. **True/False 型別的匯入正確性未經確認**：範本本身沒有區分「一般選擇題」跟「True/False」的欄位或型別標記，本 skill 把 `true_false` 轉成 2 答案選擇題——邏輯合理但未經真實匯入驗證是否會被 Kahoot 正確識別為 True/False 題型（而非顯示成只有兩個選項的一般選擇題）。
3. **圖片/媒體支援**：範本沒有圖片欄位；若 Kahoot 匯入流程其實在上傳後的編輯畫面允許逐題補圖，可以在 SKILL.md 的「上傳後請老師做的事」補充這個步驟的具體操作方式（目前只籠統提醒需要手動加圖）。
4. **官方是否有「Add question → Import → 從其他工具匯入」等其他批次路徑**（例如 Google Forms 匯入）：本輪只確認了 xlsx 試算表匯入，未調查是否有其他官方批次建立管道。
5. **Kahoot Question Bank / 題庫功能**：官方是否有可搜尋公開題庫直接引用的功能（其他平台如 Wordwall 有類似「尋找問題」側欄），本輪未調查。
