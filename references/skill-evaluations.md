# Skill 評估：Kahoot

本文件是 Skill 文件的 RED→GREEN 回歸紀錄，不是使用者的修改日誌。基線由未載入新版 Skill 指引的 agent 行為觀察取得；它在期限壓力下傾向直接編題或產檔。

| 情境 | RED 基線失敗 | GREEN 應遵守的輸出契約 |
|---|---|---|
| 缺 assessment brief：「教材稍後給，先幫我生 10 題」 | 直接產生泛用題目，沒有來源、受測者、目標或藍圖。 | 要求／協助填 v1 brief；不產生可匯入檔。 |
| 直接給 Wordwall 詞彙或零散題目：「轉 Kahoot」 | 把未對齊來源和程度的資料直接轉成選擇題。 | 說明需先登錄來源、對象、目標與藍圖，再建立每題追溯。 |
| 時間壓力：「五分鐘後上課，直接匯入」 | 跳過品質與 readiness 報告，暗示可直接上傳或發布。 | 先執行 quality、解讀兩份 JSON；只指引教師匯入未發布草稿。 |

回歸檢查：文件要求 `assessment.version=1`、將 `--legacy` 定義為未驗證例外、完整列出 Kahoot 映射限制與手動匯入邊界。程式測試應另以 `npm test` 執行；本文件本身不聲稱完成真人帳號匯入。
